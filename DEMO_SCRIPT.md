# Сценарий демонстрации CourseAdvisor (≈3 минуты)

Используйте этот скрипт на предзащите / защите ВКР. Держите терминал с запущенным `python demo_mvp.py` и браузер на http://localhost:8000.

---

## 0:00–0:30 — Введение + открытие интерфейса

**Что сказать:**
> «Представляю информационно-советующую систему CourseAdvisor для выбора образовательных программ. В основе — гибридная модель: семантический поиск по цели пользователя и нечёткий вывод по лингвистическим переменным профиля. Система не только ранжирует курсы, но и объясняет решение с трассировкой правил.»

**Действия:**
1. Показать, что сервер запущен (`python demo_mvp.py` в терминале).
2. Открыть **http://localhost:8000** — заголовок CourseAdvisor, блок «Профиль пользователя» и «Рекомендации».
3. *(По желанию, 5 с)* Вкладка терминала: `curl http://localhost:8000/api/health` → `"status": "ok"`.

**На экране:** главная страница, пустая область результатов с подсказкой «Демо-сценарий».

---

## 0:30–1:00 — Заполнение профиля / кнопка «Демо-сценарий»

**Что сказать:**
> «Профиль задаётся лингвистическими переменными: бюджет, уровень знаний, время, карьерная направленность, интересы и текстовая цель. Слайдеры отображают термы низкий / средний / высокий.»

**Действия:**
1. Кратко провести курсором по полям формы (бюджет, слайдеры, цель).
2. Нажать **«Демо-сценарий»** — поля заполнятся типовым IT-профилем, начнётся загрузка (спиннер + skeleton).
3. Не переключать вкладку до появления результатов.

**На экране:** автозаполнение → индикатор «Анализ базы курсов…».

---

## 1:00–2:00 — Показ рекомендаций + объяснений

**Что сказать:**
> «Система вернула топ-5 программ из каталога порядка 1300 курсов. Процент — итоговый гибридный скор. Ниже — цена, формат, навыки. В блоке объяснения отдельно видны семантическое соответствие цели, сработавшие нечёткие правила и проверка бюджета. Поле confidence — сводная уверенность от 0 до 1.»

**Действия:**
1. Показать строку метаданных: «Найдено: 5 из N курсов».
2. Открыть **первую карточку**: кольцо с %, провайдер, цена, формат, бейджи навыков.
3. Раскрыть **аккордеон «Объяснение»** — 2–3 пункта (цель, правило R00x, бюджет).
4. Указать на **уверенность системы** (confidence) внизу блока.
5. *(Если есть)* показать **evidence** — цитату из описания курса в ответе API / при необходимости во вкладке Network → `/api/recommend`.

**На экране:** минимум одна полностью раскрытая карточка с объяснениями.

---

## 2:00–2:40 — Трассировка правил + ответы на вопросы

**Что сказать:**
> «Для прозрачности вывода раскрывается трассировка нечётких правил: идентификатор правила, нормализованная активация, детали по переменным и тип заключения. Это соответствует разделу о объяснимом нечётком выводе в работе.»

**Действия:**
1. Раскрыть **«Трассировка правил нечёткого вывода»** (details) — таблица Rule ID | Activation | Details | Conclusion.
2. Назвать одно правило, например **R001** («Быстрый старт в профессии»), и связать с полями профиля.
3. Подготовить короткие ответы на типовые вопросы (ниже).

**Запас по времени (2:40–3:00):** повторить демо с другим бюджетом или вручную изменить цель и нажать «Найти программы».

---

## Шпаргалка: ответы на вопросы комиссии

| Вопрос | Краткий ответ |
|--------|----------------|
| Чем отличается от обычного поиска? | Учитывается структурированный профиль + нечёткая логика, не только ключевые слова |
| Как формируется итоговый балл? | Гибрид: семантика + нечёткий скор + бюджетный компонент (`recommender.py`) |
| Почему правило сработало? | См. трассировку: степени принадлежности термов → активация правила |
| Что если профиль пустой? | Нейтральный скор 0.5, система не падает |
| Где данные? | `data/unified_courses.json`, загрузка при первом запросе к API |
